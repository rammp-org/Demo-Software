   - button_push_controller.py never publishes feedback during execution
   - Simplest fix: remove the feedback field from DoorOpen.action (make feedback empty)
   - This avoids adding unnecessary publishing code for a value we don't use
   - NOTE: this changes the interface — rebuild cmu_door_opener_interfaces after

   - button_push_controller.py subscribes to `/arm/joint_states` (JointState)

   - Delete the subscriber creation and the callback method

6. ADD AGE VALIDATION FOR BUTTON_INFO MESSAGES (10s threshold)

     alongside the message (e.g., self.latest_button_time = self.get_clock().now())

     check age against 10 second threshold
   - If older than 10s, abort the action with a "stale button data" error
   - ButtonInfo.msg has no header — so we timestamp on arrival
